1.3.2

What's new?

  • PRICING: In case our main source of pricing fails, the extension now falls back to the pricing API by Steaminventoryhelper, which is also getting cached for the duration of a session to reduce network load. Overall, this should improve reliability of pricing without any overhead.
  • SITES: The 'unique items' tab is now also supported by API-based features such as the sticker pricing.
1.3.1

What's new?

  • CHANGE: The 10s interval option of the auto-refresh feature has been removed due to unjustifiably long rate limits after very short periods of usage. For now it is unclear if rate-limits by CSFloat can long term lead to account bans.
  • BUG-FIX: Additional error handling if the data provided by the CSFloat API contains errors or wrong information. Fixes an issue that previous led to a crash when CSFloat supplied a sticker name with wrong capitalization.
  • BUG-FIX: A long-time bug of the stop button not actually stopping the refreshing due to a data race should now be fixed.

What's wrong?

  • Both the niko and NiKo stickers currently lead to errors due to their only difference being capilization. This may lead to such stickers not linking to their Buff page or missing in sticker price calculations. Possible fixes are already getting investigated and will be included in a future version.
1.3.0

What's new?

  • FEATURE: Back to top button: when scrolling down, a "back to top"-button will now appear in the bottom right corner. On click, you will smoothly get back to the top of the page.
  • FEATURE: Item popups now display the highest/lowest recorded past price above its history graph. This price always refers to the daily average, as this is the only data available on CSFloat. As I believe that this can never negatively impact the user's experience, this is an always-on feature for now.
  • SETTING: A new setting allows for variation of the content in the sale-tag next to a listings price. The options are to either keep the original price difference to the Steam price in percentage or see the price difference to the Buff price in absolute values. Default value is Buff.
1.2.1

What's new?

  • FIX: Sticker price feature now also works in stalls.
  • TECH-STACK: With this update, the codebase has been adapted to be compatiable with and use the Typescript option "strict: true". Overall this will lead to massive improvements in stability, consistency and improvability for all features and functionality.
1.2.0

What's new?

  • FEATURE: Show listing age → Show the listing age in the listing card. The two available options determine the position of the text. This setting is per default turned off to prevent cluttering in standard use cases.
  • BUG-FIXING: Hopefully any isses with case-sensitive OS are fixed now by reverting any capitalization in file names. The original issue is explained below.

What's wrong?

  • Since version 1.1.0 case-sensitive OS (e.g. Ubuntu, Android) did occasionally print errors due to the changed capitalization of file names. Non-case-sensitive OS (e.g. Windows) have not been influenced by this at all. Please send a report to our Discord channel for further debugging if this is the case for your system.
1.1.0

What's new?

  • FEATURE: Sticker Prices → A new tag above the applied stickers now displays a calculated sticker price percentage. This SP% is depicted by the ratio of the item price difference to the total buff price (ask) of all stickers. This feature can be disabled in the popup (default: on).
  • POPUP: Completely reworked the logic behind the popup. It now also features a 'changelog' tab displaying the latest changes.
  • BACKGROUND: The extension is now able to detect and make use of the CSFloat API requests already sent by the website. This will allow many new possibilities in the future.
  • TECH-STACK: Building and deploying the extension is now done using webpack. This allows for a much cleaner codebase and easier development. Please refer to the new README for more information.

What's wrong?

  • If you have built the extension manually in the past, you may have to re-install the new one manually. You may want to switch to the officially published version on the Chrome Web Store or the Firefox Add-Ons Store instead.
